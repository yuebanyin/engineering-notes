# 写作规范

本文档定义知识库的写作风格与规范。

## 核心原则

1. **结构清晰**：每篇笔记遵循"问题 → 约束 → 决策 → 验证"的结构
2. **实用导向**：优先记录可复用的模式和决策过程
3. **非机密**：不包含任何专有代码或敏感信息

## 标题规范

- 使用动词或名词短语
- 简洁明了，一眼看懂主题
- 示例：
  - ✅ React 渲染性能优化清单
  - ✅ 如何设计可扩展的 API 错误处理
  - ❌ 关于一些性能的想法

## 内容规范

### 代码片段
- 仅包含通用、非敏感的代码
- 必须可运行或注明伪代码
- 包含必要的上下文说明

### 引用
- 优先引用官方文档
- 注明版本号（如适用）
- 使用标准 Markdown 链接格式

## 文件命名

- 使用小写字母和连字符
- 示例：`react-rendering-model.md`
- 目录索引文件统一命名为 `README.md` 或 `00-roadmap.md`

## 更新频率

- 新增笔记后更新 `INDEX.md`（如果是代表作）
- 重要更新记录在"最近更新"部分
