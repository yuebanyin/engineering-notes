# 一句话理解每个关键词

| 关键词              | 前端类比                     | 实际含义                                                                 |
| ------------------- | ---------------------------- | ------------------------------------------------------------------------ |
| **SWE-bench**       | 就像 LeetCode 的题库         | 从 GitHub 真实 issue 收集的"修 bug 考题"，用来考 AI 能不能修代码         |
| **SWE-smith**       | 就像自动出题机器             | 能给任何 GitHub 仓库自动生成"考题"，产出 5 万多道题 + 对应的 Docker 环境 |
| **Terminal-Bench2** | 就像考 Linux 运维的题库      | 考 AI 能不能在终端里执行命令（装软件、改权限、查网络等）                 |
| **IT_Bench**        | 就像考高级运维的题库         | 考 AI 能不能排查复杂故障（502 错误、数据库挂了等多系统联动问题）         |
| **Build Image**     | 就像 npm install + npm build | 把"考试环境"打包成 Docker 镜像（题目 + 依赖 + 初始状态）                 |
| **Sanity Check**    | 就像 npm test                | 跑一遍验证：这道题本身是不是有效的（答案对不对、环境能不能跑）           |
| **Pipeline (ADF)**  | 就像 CI/CD (GitHub Actions)  | 把 build + test 自动化，批量跑几百道题，不用手动一个个来                 |
| **CaaS**            | 就像 CodeSandbox             | 远程容器服务，启动一个容器 → 跑命令 → 拿结果 → 销毁                      |
| **ACR**             | 就像 npm registry            | 存 Docker 镜像的仓库（你之前迁移的 222 个镜像就放在这里）                |
